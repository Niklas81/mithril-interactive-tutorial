The Controller decides what data in our Model should be available to the View. It may optionally manipulate that data as it passes through. The Controller can also pass input coming from the View back to the Model. In the next lesson we'll learn to simplify our controller.

Bi-Directional Data Flow

Our controller is responsible for passing data back and forth between the model and the view. The add method must keep the information in the view in sync with the data in our model. The input value must be set when the Add button is clicked. We handle this by setting the input value to the result of calling our getter/setter todo.description(). The Model must be updated as well. This requires the mithril function m.withAttr() which is explained thoroughly in the comments in the source code.
