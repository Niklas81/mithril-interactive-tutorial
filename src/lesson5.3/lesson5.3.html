The Controller decides what data in our Model should be available to the View. It may optionally manipulate that data as it passes through. The Controller can also pass input coming from the View back to the Model. In the next lesson we'll learn to simplify our controller.

Data Binding

Our controller is responsible for passing data back and forth between the model and the view. The ctrl.add method takes a description from the text input, instantiates a new todo.Todo with that description, and adds it to our todo.TodoList.

The input "value" attribute is set when the Add button is clicked. We handle this by binding "value" to the result of ctrl.description(). After ctrl.add pushes a new todo on the list, it resets the description to an empty string. Since the input "value" attribute is bound to the description getter/setter, it becomes an empty string as well. The input text area is blank again.
