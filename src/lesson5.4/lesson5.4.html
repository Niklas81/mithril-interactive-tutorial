The Controller should connect the Model with the View but shouldn't know anything about the inner workings of either. Our controller is too tightly coupled with the View and Model, so now we factor out our controller logic to the companion to our data model: the view-model.

src/components/Todo5.4.coffee

There are many definitions of the MVC paradigm. The role of the Model might be the most controversial. One approach is to make the model the single source of authority for all data state and view state. Our model will have these responsibilities:

  • Store data in memory
  • Maintain the state of the data
  • Receive data from the UI
  • Maintain the state of the UI
In a real world scenario, data would be served-from and save-to some sort of database on a server. We've already learned how to get json data with m.request(). We'll learn to save data to the server in a later lesson. The view-model would be resposible for making this happen.
