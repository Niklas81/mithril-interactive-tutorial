Our Todo List begins life with no tasks. Input will come from the user through the View and we'll store new tasks in a Model.

src/lesson5.2/Todo5.2.coffee

The Model defines two classes:

  • todo.Todo -- individual tasks with a desctiption and a "done" switch
  • todo.TodoList -- an array of Todos for display in the View

Each instance of the Todo class will store a description (string) of one task.

Beyond storing data, the Model is also responsible for managing component state. In our case, we need the Model to keep track of the "done" state of each Todo. When the user checks a task, "done" is set to true.

done("true")

A boulean switch like done/not-done is a perfect case for using an m.prop getter/setter. An input checkbox both sets and display the "done" state.

The TodoList is where we will store Todo instance objects added when the user adds a new task.
