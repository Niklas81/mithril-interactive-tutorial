The Getting Started page of the Mithril documentation teaches you how to build a classic todo list. This lesson will show you a working Todo List component that you can modify and experiment with.

This lesson is composed of a series of steps on individual pages. We will learn to make Single Page Apps in a later lesson, but sometimes separate pages are the best choice. The Mithril home page is a typical website with individual pages, yet Mithril components are scattered throughout.

The components for this lesson are reused on each page.

src/app.coffee
  • Logo
  • Header
  • Content
  • Todo
  • Footer
The Header and Footer are written so that defaults can be overridden to individualize each page.

src/Todo5.coffee

User experience is defined as follows:

  • User enters task description and clicks the Add button
  • A list of added tasks appears below the input
  • User checks "done" and task is marked with a line through

We need to build functionality to fulfill each UX requirement. This is a working example of our Todo List component. Try it out!
