The Getting Started page of the Mithril documentation teaches you how to build a classic todo list. This lesson will show you a working Todo List component that you can modify and experiment with.

This lesson is composed of a series of steps on individual pages. Each page is a complete Mithril application, but static. In a later lesson we will learn to build Single Page Apps (SPA), allowing us to morph one page into endless configurations. But sometimes we want separate pages with components mixed in. The Mithril home page is a typical website with individual pages, yet Mithril components are scattered throughout.

The components for this lesson are reused on each page. They do not persist in the browser, they are re-loaded with each page change. They are reused in the sense that the components can be customized to individualize each page.

The Logo, Header and Footer are written so that defaults can be overridden to individualize each page. Content is just static HTML. Todo is our example app. We've also nested a Menu component inside the Footer. See Footer.coffee.

src/Todo5.coffee

Our intended user experience is defined as follows:

  • User inputs task description and clicks Add button
  • List of added tasks appears below input
  • User checks "done" and task is marked with line through

We need to build functionality to fulfill each UX requirement. This is a working example of our Todo List component. Try it out!
